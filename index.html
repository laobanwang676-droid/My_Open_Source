边缘 AI 的低功耗智能健康监护穿戴设备

一、项目概述

本项目以STM32U575VGT6低功耗主控芯片为核心，搭载 FreeRTOS 实时操作系统，打造一款面向健康监护场景的便携式可穿戴设备。系统深度融合边缘 AI 推理、多体征传感器采集、4G 无线通信、GPS 定位、远程 OTA 升级等核心技术，实现跌倒智能检测、心率血氧实时监测、低电量提醒、紧急告警、定位上报、云端数据互联等一体化功能。设备采用极致低功耗设计搭配流畅触控界面，兼顾长时间续航与良好人机体验，是集合嵌入式低功耗开发、边缘 AI 运算、物联网通信、固件升级技术于一体的综合性智能硬件项目。

二、硬件平台

  • 主控芯片：STM32U575VGT6，Cortex-M33 内核，具备优异低功耗特性，满足边缘计算运行需求
  • 姿态采集：SC7A20H 三轴加速度计，负责人体运动姿态原始数据采集
  • 体征检测：MAX30102 心率血氧传感器，精准采集人体心率、血氧饱和度参数
  • 定位模块：ATGM336H GPS 定位模组，实现户外精准位置获取
  • 无线通信：Air780E 4G 模块，承担数据上传、短信告警、远程 OTA 通信任务
  • 人机交互：电阻触摸屏 + 液晶显示屏，基于 LVGL 实现可视化界面操作
  • 电源管控：NMOS 低端开关电路，按需通断外设供电；搭载电量检测电路
  • 存储介质：片内 Flash，用于本地配置数据持久化保存

三、核心技术与运行机制

1. 操作系统与任务调度
基于 FreeRTOS 实现多任务并行管理，通过互斥锁、消息队列解决多任务资源竞态与程序阻塞问题，稳定调度体征采集、AI 跌倒判别、网络上传、界面刷新、电量监测等任务，保障系统实时响应。

2. 边缘 AI 跌倒检测
SC7A20H 三轴加速度计不间断采集运动数据，提取轻量化统计特征与峰值特征；依托小型全连接神经网络完成行为判别，同时结合瞬时加速度差分尖峰阈值做联合决策，有效提升跌倒识别准确率，适配嵌入式设备算力与功耗限制。

3. 物联网数据交互
整合 ATGM336H 定位模块与 Air780E 4G 模组，采集到的心率血氧、定位坐标数据，通过 MQTT 协议上传至 OneNET 云平台；设备判定跌倒险情后，自动发送告警短信并同步位置信息至紧急联系人，实现远程监护预警。

4. 高性能触控界面
采用 LVGL 图形框架搭建多页面交互系统，涵盖主页、血氧检测、GPS 定位、联系人设置、摔倒提醒、OTA 升级确认六大功能界面。运用分散加载技术，将 LVGL 核心渲染程序、高频运算函数重定位至 SRAM 中运行，有效提升界面刷新速度与图形操作流畅度。

5. 安全远程 OTA 升级
借助 HTTP 通信协议实现设备在线固件更新，升级文件通过 MD5 算法完整性校验，规避固件损坏导致设备无法正常工作的风险，方便后期功能迭代维护。

6. 本地数据持久化
利用芯片片内 Flash 读写存储紧急联系人号码，支持触摸屏自定义编辑保存参数，解决设备重启后配置丢失的问题。

7. 低功耗与电量管理
通过 NMOS 低端开关按需切断闲置外设供电，搭配一键息屏、点击亮屏的节能策略缩减待机损耗；实时检测设备剩余电量，界面直观展示电量状态，电量低于阈值时自动触发低电量报警提示。

四、核心功能

  • 生命体征监测：MAX30102 传感器实时采集心率、血氧数据，本地显示并同步上传云端存档
  • 边缘 AI 跌倒检测：多算法联合判定人体跌倒动作，识别灵敏、误判率低
  • 定位与紧急告警：ATGM336H 获取精准位置，险情发生后 4G 模块推送短信与定位信息
  • 云端数据互通：MQTT 协议对接 OneNET 云平台，支持远程查看历史监测数据
  • 触控人机交互：多场景功能页面自由切换，支持参数自定义配置操作
  • 安全 OTA 升级：远程更新设备固件，MD5校验 + 回滚双重机制保障升级稳定
  • 本地配置存储：联系人信息断电不丢失，可随时触屏修改保存
  • 智能功耗控制：外设智能断电、屏幕息屏节能，延长设备使用时长
  • 电量监测报警：实时显示剩余电量，电量不足时触发本地低电量警示

五、关键难点与解决方案

设备功耗偏高，难以满足穿戴设备续航要求
难点：传感器、4G、GPS 外设长期通电，待机损耗大；解决方案：采用 NMOS 低端开关单独管控外设供电，闲置设备自动断电；增设息屏唤醒功能，最大程度降低待机功耗。

用户配置无法保存，重启后联系人参数重置
难点：断电后自定义号码丢失，每次开机需重新设置；解决方案：读写片内 Flash 完成数据持久化，触屏编辑后自动保存配置，断电重启数据不丢失。

多任务并发引发资源争夺、程序卡顿阻塞
难点：数据采集、AI 运算、网络传输、界面刷新同步运行，易出现数据错乱；解决方案：使用 FreeRTOS 互斥锁管控共享资源，依靠消息队列完成任务间数据传递，规避竞态问题。

图形界面刷新缓慢，触控交互存在延迟
难点：Flash 中运行图形渲染代码运行效率偏低；解决方案：采用分散加载方式，将高频执行的界面渲染函数迁移至 SRAM 运行，大幅提升界面流畅度。

电池电量状态无法直观把控，低电量易突发断电
难点：无法实时获知剩余电量，电量耗尽影响设备正常监护；解决方案：增加电量检测电路，界面实时展示电量百分比，到达临界值主动发出低电量报警提醒。

六、项目亮点

  • 低功耗穿戴级设计：STM32U5 低功耗主控搭配智能电源管控，续航表现优异，适配日常穿戴使用
  • 端侧 AI 实用落地：轻量化神经网络结合阈值判定，本地完成跌倒识别，无需依赖云端运算
  • 全链路物联网体系：Air780E+ATGM336H 模组搭配 MQTT 云平台，实现监测、定位、告警全流程远程联动
  • 可靠固件迭代方案：HTTP 远程升级搭配 MD5 校验，保障设备后期维护安全稳定
  • 高性能图形交互：SRAM 加速界面渲染，嵌入式设备实现顺滑触控操作体验
  • 系统运行稳定可靠：RTOS 任务调度配合通信同步机制，多任务运行无死机、数据无异常
  • 完备状态监测体系：兼具体征监测、运动判别、电量预警，全方位守护使用安全

七、应用场景

本项目适用于独居老人健康监护、儿童出行防护、慢性病日常监测、个人随身安全防护等场景，全面覆盖低功耗单片机开发、边缘 AI 算法、4G 物联网通信、GPS 定位、RTOS 操作系统、LVGL 图形开发、固件 OTA 升级等多项嵌入式核心技术，具备较高实战与项目实践价值。

基于FreeRTOS的智能天气时钟

一、项目概述

本项目是一款多功能嵌入式智能终端系统，以 STM32F407VET6 为主控核心，搭载 FreeRTOS 实时操作系统实现多任务协同运行，系统兼容 STM32 标准库与 HAL 库双版本开发，具备极强的可移植性与维护性。系统深度融合环境感知、网络数据交互、语音智能交互、实时报警与可视化人机交互等核心能力，实现了从数据采集、网络通信到智能响应、本地显示的全流程闭环，是一套完整的嵌入式综合应用解决方案。

二、硬件与技术栈

核心硬件

  • 主控芯片：STM32F407VET6（高性能 ARM Cortex-M4 内核）
  • 通信模块：ESP32C3 模块（WiFi 联网、网络数据获取）
  • 传感模块：AHT20 I2C 接口温湿度传感器
  • 显示模块：ST7789 LCD 显示屏（SPI 接口 + DMA 传输）
  • 交互模块：SU03T语音识别 / LU6288播报模块、物理按键

核心技术

  • 操作系统：FreeRTOS（任务调度、进程间通信）
  • 驱动框架：LVGL 嵌入式图形库
  • 通信协议：UART 串口、I2C、SPI、DMA
  • 软件兼容：STM32 标准库 / HAL 库双版本支持

三、系统架构与核心机制

系统采用FreeRTOS 多任务架构设计，通过精细化的任务调度实现各功能模块并行运行；依托队列、信号量等 IPC 机制完成任务间数据同步与资源互斥访问，保障系统实时性与稳定性。主控通过UART 串口与 ESP32 通信，结合串口中断接收 + 指令校验机制，确保天气数据、网络时间等网络信息的精准获取；通过I2C 协议与温湿度传感器通信，实现环境数据实时采集；采用LVGL 图形框架驱动 LCD，搭配SPI+DMA数据传输方式，大幅提升屏幕刷新效率，降低 CPU 占用率；支持按键配网、语音识别、阈值报警等智能化交互逻辑，打造一体化智能体验。

四、核心功能

  • 智能环境监测：实时采集环境温湿度数据，支持自定义阈值，触发声光 / 界面智能报警；
  • WiFi 网络交互：按键快速配网，通过 ESP32 获取公网天气数据、网络标准时间；
  • 语音智能交互：支持语音指令识别，可播报温湿度、天气、网络时间等信息；
  • LVGL 人机交互：可视化界面实时展示环境数据、网络状态、报警信息，操作便捷；
  • 高可靠通信：串口指令校验、数据容错处理，保障模块间通信稳定可靠。

五、关键难点与解决方案

LCD 多任务数据显示竞态问题
难点：多个任务同时操作显示屏，易出现数据错乱、显示异常；
解决方案：采用 FreeRTOS 队列作为数据传输中介，统一管理显示数据，避免多任务资源竞争，保证显示流畅性与数据准确性。

串口数据接收可靠性问题
难点：ESP32 高频数据传输易出现丢包、误码；
解决方案：使用串口中断接收数据，搭配指令校验机制，确保网络数据解析精准。

多外设资源冲突问题
难点：传感器、WiFi、显示屏、语音模块共用系统资源；
解决方案：通过信号量实现资源互斥访问，FreeRTOS 优先级调度保障核心任务优先执行。

六、项目亮点

  • 双库兼容设计：同时支持标准库与 HAL 库开发，适配不同开发习惯与移植需求；
  • 实时性强：基于 FreeRTOS 的多任务架构，响应速度快、系统稳定性高；
  • 高效显示：SPI+DMA+LVGL 组合，实现高帧率、低功耗的人机界面；
  • 功能模块化：各功能解耦设计，便于二次开发、功能扩展与调试维护；
  • 全场景交互：融合按键、语音、可视化界面，打造沉浸式智能交互体验。

七、应用场景

本系统可广泛应用于智能家居环境监测、室内智能终端、嵌入式教学实训、工业环境监控节点等场景，是嵌入式 RTOS 开发、外设驱动、通信协议、人机交互等技术的综合实践项目。

多模态两轮自平衡系统

一、项目概述

本项目以STM32F103C8T6为主控芯片，采用标准库与 HAL 库双版本工程开发，打造一款高性能两轮自平衡智能小车。系统融合姿态闭环平衡、自主循迹、智能避障、蓝牙远程操控四大核心功能，通过多传感器数据融合与高精度闭环控制算法，实现稳定的自主运行与远程交互，是嵌入式控制、传感器融合、运动控制算法的综合实践项目。

二、硬件平台

  • 主控芯片：STM32F103C8T6（ARM Cortex-M3 内核，高性能低成本）
  • 姿态传感器：MPU6050（六轴陀螺仪 + 加速度计，用于姿态解算）
  • 执行机构：双路 520 直流减速电机，驱动小车运动与平衡
  • 显示模块：0.96 寸 OLED 显示屏，实时显示小车姿态、运行模式、传感器数据
  • 通信模块：蓝牙串口模块，实现手机 APP 远程无线操控
  • 环境传感器：HC-SR04 超声波模块、五路红外循迹模块
  • 保护机制：倒地检测、过流保护、扶正自启动硬件电路

三、核心技术与控制机制

姿态感知与解算
通过软件 I2C 协议读取 MPU6050 原始数据，结合卡尔曼滤波算法对陀螺仪与加速度计数据进行融合解算，输出高精度实时姿态角。

三级闭环 PID 控制
搭建直立环 + 速度环 + 转向环三环 PID 控制架构：直立环保证小车动态平衡，速度环实现匀速行驶，转向环完成精准转向控制，三者协同保障小车稳定运行。

智能感知控制
五路红外循迹模块采集路面信号，采用 PD 闭环控制实现高精度黑线循迹；HC-SR04 超声波实时检测前方障碍物，触发自动避障策略；蓝牙与 OLED 提供交互与状态显示。

四、核心功能

  • 自平衡控制：基于三环 PID 实现小车动态直立平衡，抗干扰能力强；
  • 自主循迹：五路红外 + PD 控制，精准识别黑线轨迹，稳定沿迹行驶；
  • 智能避障：超声波实时测距，自动绕行障碍物，无碰撞运行；
  • 蓝牙遥控：手机 APP 远程控制小车启停、转向、调速；
  • 状态显示：OLED 实时输出姿态角、运行模式、传感器数据；
  • 安全保护：倒地自动停机，扶正后快速恢复平衡状态。

五、关键难点与解决方案

循迹精度低、行驶抖动
难点：传统开关控制循迹偏差大、响应迟钝；
解决方案：采用 PD 闭环控制算法，优化响应速度与稳定性，循迹平滑度大幅提升。

MPU6050 姿态解算零漂严重
难点：陀螺仪温漂、加速度计噪声导致姿态角漂移；
解决方案：引入卡尔曼滤波融合多传感器数据，有效抑制零漂，提升平衡稳定性。

多任务控制冲突
难点：平衡、循迹、避障多任务并行导致控制紊乱；
解决方案：模块化任务调度，优先级分配核心控制任务，保证系统实时性。

六、项目亮点

  • 双库兼容开发：同时支持标准库与 HAL 库，适配不同开发场景；
  • 多模式一体化：集自平衡、循迹、避障、遥控于一体；
  • 高精度算法：卡尔曼滤波 + 三环 PID + PD 控制，控制精度高；
  • 可视化交互：OLED + 蓝牙远程操控，人机交互友好；
  • 高安全性：完善的硬件保护与软件容错机制。

七、应用场景

本项目可应用于嵌入式运动控制教学实训、智能小车竞赛、科创 DIY 项目、自主移动机器人入门开发等场景，覆盖传感器驱动、滤波算法、PID 控制、无线通信等嵌入式核心技术点。
